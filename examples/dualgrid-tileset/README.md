# 双网格（Dual-Grid）瓦片系统 — 实现文档

## 概述

本系统实现了一套基于**双网格**（Dual-Grid）算法的 2D 地形瓦片生成管线，全部在 PML（Picture Markup Language）中完成。核心思路是将**数据网格**和**渲染网格**偏移半个瓦片，通过采样四个角的地形类型计算 4-bit bitmask，从而确定每个渲染像素属于哪种地形及其过渡形状。

### 输出文件

| 文件 | 尺寸 | 说明 |
|------|------|------|
| `00_spritesheet.png` | 2048×512 | 完整精灵表：4 种地形 × 16 种过渡 = 64 个瓦片 |
| `01_demo_tilemap.png` | 640×640 | 演示地图：草地中嵌入泥地 |
| `02_cross_terrain.png` | 640×640 | 多地形交叉演示：水域、泥地、石头、草地 |

### 地形类型

| ID | 名称 | 颜色范围 |
|----|------|----------|
| 0 | Grass（草地） | 深绿 → 亮绿 |
| 1 | Dirt（泥地） | 深褐 → 亮褐 |
| 2 | Stone（石头） | 深灰 → 浅灰 |
| 3 | Water（水域） | 深蓝 → 亮蓝 |

### 16 种过渡模式（4-bit bitmask）

每个瓦片对应数据网格中四个角的位置，每个位置取 0 或 1 表示是否为目标地形：

```
bitmask = (BL << 0) | (BR << 1) | (TL << 2) | (TR << 3)
```

| 索引 | 模式 | BL | BR | TL | TR | 形状描述 |
|------|------|----|----|----|----|----------|
| 0 | EMPTY | 0 | 0 | 0 | 0 | 全部为背景 |
| 1 | BL | 1 | 0 | 0 | 0 | 左下角 1/4 圆 |
| 2 | BR | 0 | 1 | 0 | 0 | 右下角 1/4 圆 |
| 3 | BL+BR | 1 | 1 | 0 | 0 | 下半平直过渡 |
| 4 | TL | 0 | 0 | 1 | 0 | 左上角 1/4 圆 |
| 5 | TL+BL | 1 | 0 | 1 | 0 | 左半平直过渡 |
| 6 | TL+BR | 0 | 1 | 1 | 0 | 对角双圆（左上+右下） |
| 7 | TL+BL+BR | 1 | 1 | 1 | 0 | 上半凹 1/4 圆 |
| 8 | TR | 0 | 0 | 0 | 1 | 右上角 1/4 圆 |
| 9 | TR+BL | 1 | 0 | 0 | 1 | 对角双圆（右上+左下） |
| 10 | TR+BR | 0 | 1 | 0 | 1 | 右半平直过渡 |
| 11 | TR+BL+BR | 1 | 1 | 0 | 1 | 左半凹 1/4 圆 |
| 12 | TL+TR | 0 | 0 | 1 | 1 | 上半平直过渡 |
| 13 | TL+TR+BL | 1 | 0 | 1 | 1 | 右下凹 1/4 圆 |
| 14 | TL+TR+BR | 0 | 1 | 1 | 1 | 左下凹 1/4 圆 |
| 15 | FULL | 1 | 1 | 1 | 1 | 全部为目标地形 |

## 架构概览

```
SkSL Shader（自包含着色器）
├── FBM 噪声函数（hash → fbm3 → fbm2）
├── 域扭曲（domain warping）
├── 地形着色（terrain_color）
├── SDF 评估（有符号距离场）
├── 混合逻辑（背景/目标地形过渡）
└── 输出像素

PML 驱动层
├── 精灵表生成（4×16 网格）
├── 演示地图生成（5×5 渲染网格）
└── 多地形演示（6×6 数据网格）
```

## 详细实现流程

### 1. 着色器核心：SkSL 自包含

所有的地形生成逻辑都封装在一个自包含的 SkSL（Skia Shader Language）着色器中，通过 PML 的 `shader` 关键字传入字符串。着色器接收 6 个 uniform 参数：

```
u_bl, u_br, u_tl, u_tr  → 四个角的 bitmask 值（0.0 或 1.0）
u_seed                    → 随机种子（每个瓦片不同）
u_type                    → 地形类型 ID（0.0~3.0）
```

#### 1.1 哈希函数

```glsl
float hash(float2 p) {
    return fract(sin(dot(p, float2(127.1, 311.7))) * 43758.5453123);
}
```

标准的伪随机函数，输入 2D 坐标，输出 [0,1) 的哈希值。用于 FBM 噪声的基础。

#### 1.2 FBM（分形布朗运动）

实现了两个版本：

- **`fbm3`** — 3 个八度的 FBM（用于域扭曲）
- **`fbm2`** — 2 个八度的 FBM（用于地形纹理）

由于 Skia 不支持 `for` 循环（`for (int i; ...)` 会报错），每个八度都手动展开为独立的代码块。每个八度的步骤：

```
1. 计算 floor(p) → 整数网格坐标 a
2. 计算 fract(p) → 小数偏移 b
3. 对 b 应用平滑步进：b = b * b * (3 - 2 * b)
4. 在四个整数角采样 hash，用 b 做双线性插值
5. 累加：v += 权重 * mix(...)
6. p *= 2.0（下个八度频率翻倍）
```

权重递减：0.5 → 0.25 → 0.125，实现频率加倍、振幅减半的标准 FBM。

#### 1.3 域扭曲（Domain Warping）

为了产生有机感更强的地形边缘，在计算每个像素的颜色之前，先用 FBM 对 UV 坐标进行扭曲：

```glsl
float2 wuv = xy * 0.025 + u_seed;
float  wx  = (fbm3(wuv) - 0.5) * 16.0;
float  wy  = (fbm3(wuv + 60.0) - 0.5) * 16.0;
float2 wxy = clamp(xy + float2(wx, wy), 0.0, ts);
```

`ts = 128.0` 是瓦片尺寸。扭曲幅度最大 ±8 像素，使纯粹的数学 SDF 边界产生自然起伏。

#### 1.4 地形着色器

```glsl
half3 terrain_color(float2 uv, float seed, float type)
```

根据 `type` 参数选择地形颜色方案，使用 `fbm2` 生成纹理细节：

- **草地（type < 0.5）**：深绿(0.10,0.30,0.08) ↔ 亮绿(0.25,0.65,0.20)
- **泥地（type < 1.5）**：深褐(0.30,0.18,0.08) ↔ 亮褐(0.65,0.50,0.30)
- **石头（type < 2.5）**：深灰(0.25,0.30,0.35) ↔ 浅灰(0.55,0.58,0.62)
- **水域（type >= 2.5）**：深蓝(0.05,0.15,0.40) ↔ 亮蓝(0.15,0.60,0.90)

每种地形使用两个频率的 FBM 叠加，`n`（主频，`uv * 0.03`）控制大尺度变化，`n2`（次频，`uv * 0.06`）添加细部纹理。

#### 1.5 SDF 过渡评估

每个瓦片的四个角对应四个圆的 SDF（有符号距离场）：

```
d_bl = length(xy - (0,  ts)) - r    // 左下角，圆心在左下
d_br = length(xy - (ts, ts)) - r    // 右下角，圆心在右下
d_tl = length(xy - (0,   0)) - r    // 左上角，圆心在左上
d_tr = length(xy - (ts,  0)) - r    // 右上角，圆心在右上
```

`r = 64.0` 是圆的半径（即瓦片尺寸的一半）。

根据 4 个 uniform 的 bitmask，计算最终的 `fill` 值（负值 = 目标地形内部，正值 = 外部）：

**单个角激活**：取对应角 SDF 的 `min`（即最接近的激活角）

```
if (bl) fill = min(fill, d_bl);
```

**相邻双角激活（平直线）**：用有符号距离到对应边的距离

```
BL+BR → fill = (ts - r) - xy.y    // 下半边
TL+TR → fill = xy.y - r            // 上半边
TL+BL → fill = xy.x - r            // 左边
TR+BR → fill = (ts - r) - xy.x    // 右边
```

**三个角激活**：用未激活角的 SDF 负值（凹 1/4 圆）

```
if (!bl) fill = -d_bl;
```

**全空**：`fill = 1.0`（全部为背景）
**全满**：`fill = -1.0`（全部为目标地形）

**对角双角**：使用各自 SDF 的 `min`（自然的两个分离圆）

最后通过 `smoothstep(-sw, sw, fill)` 生成 0~1 的混合因子，`sw = 4.0` 控制过渡带宽度。

#### 1.6 最终混合

```glsl
half4(gcol * (1 - a) + tcol * a, 1.0)
```

- `gcol` = 背景地形颜色（type=0，草地）
- `tcol` = 目标地形颜色（由 `u_type` 指定）
- `a` = SDF 混合因子

### 2. PML 驱动层

#### 2.1 `tile` 辅助函数

```pml
(defn tile (x y bl br tl tr seed type-id)
  (define ud (make-uniforms bl br tl tr seed type-id))
  (define s  (apply-uniforms sksl ud))
  (define go (apply-shader! (rect 0 0 TS TS :fill "#fff") s))
  (add (with-transform go (translate x y))))
```

关键 PML 特性：
- **`make-uniforms`** + **`apply-uniforms`**：将 6 个 float 打包为一个 uniform 数据块传递给 SkSL 着色器（替代逐个 `uniform-float` 多次调用的方式，修复了内存对齐问题）
- **`apply-shader!`**：将着色器应用到矩形上
- **`with-transform`** + **`translate`**：将瓦片放置到画布的正确位置

#### 2.2 精灵表生成（4 种地形 × 16 种模式）

画布尺寸：`2048 × 512`（= 16×128 × 4×128）

```pml
(canvas (* 16 TS) (* 4 TS) :bg "#1a1a2e")
```

两层嵌套循环：
- 外层：遍历 4 种地形（`tidx = 0..3`）
- 内层：遍历 16 种过渡模式（`pidx = 0..15`）

每个瓦片的坐标计算：
```
x = tidx * 4 * TS + (pidx % 4) * TS
y = (pidx / 4) * TS
```

每行右侧添加地形名称标签，顶部添加模式描述标签。

#### 2.3 演示地图生成

**草地中嵌入泥地**：5×5 渲染网格

定义每个渲染像素的模式索引：
```
2   3   3   3   1
10  15  15  15  5
10  15  15  15  5
10  15  15  15  5
8   12  12  12  4
```

- 外部：草地边缘（模式 2, 3, 1 等，SDF 过渡到背景）
- 内部：泥地（模式 15 = 全满）

#### 2.4 多地形交叉演示

**6×6 数据网格 → 5×5 渲染网格**，包含三种地形的过渡：

```
G  G  G  G  G  G
G  W  W  D  D  G    G=Grass(0), W=Water(3)
G  W  W  D  D  G    D=Dirt(1),   S=Stone(2)
G  D  D  S  S  G
G  D  D  S  S  G
G  G  G  G  G  G
```

每个渲染网格单元包含 `(type-id, pattern-idx)` 二元组，使得每个像素能正确显示两种不同地形之间的过渡。

### 3. 双网格算法解析

传统瓦片系统在数据网格的每个格子中存储地形类型，然后在格子边界处使用预定义的过渡瓦片。双网格方法将渲染网格相对于数据网格偏移半个瓦片：

```
数据网格（6×6）：
  G  G  G  G  G  G
  G  W  W  D  D  G
  G  W  W  D  D  G
  G  D  D  S  S  G
  G  D  D  S  S  G
  G  G  G  G  G  G

渲染网格（5×5）：每个像素采样数据网格的 4 个角
  (G,G,G,W) (G,G,W,W) (G,W,W,D) ...
  ...
```

每个渲染像素从数据网格的四个角采样地形类型，与当前像素的目标地形比较生成 4-bit bitmask，然后用 SDF 计算出自然的过渡形状。这就是为什么 16 种模式就足以表达任意地形之间的平滑过渡。

## 尝试验收

### 生成的图像

1. **`00_spritesheet.png`**：完整的地形瓦片库。行 = 地形类型（草地、泥地、石头、水域），列 = 16 种过渡模式。可用于游戏引擎的瓦片地图。

2. **`01_demo_tilemap.png`**：5×5 演示地图，草地背景中嵌入泥地区域。展示 SDF 边缘的自然过渡。

3. **`02_cross_terrain.png`**：5×5 多地形演示，展示三种地形（水域→泥地→石头）在同一个地图中的共存和过渡。

### 验证要点

- 过渡边缘应平滑而非像素化（`smoothstep` + `sw=4.0` 过渡带）
- 边缘应有机起伏而非平直几何（域扭曲 ±8px）
- 每种地形的纹理应有自然细节（双频率 FBM）
- 所有 16 种过渡模式应正确显示（精灵表中的 16 列逐一对应 bitmask）

## 潜在改进方向

- **增加域扭曲幅度**：将 `* 16.0` 改为 `* 24.0` 或更大值使边缘更不规则
- **更多地形类型**：扩展到沙漠、雪地、森林等（修改 `terrain_color` 新增分支）
- **自定义过渡带**：将 `sw=4.0` 改为可配置的 uniform
- **SkSL 循环优化**：如果未来 Skia 支持 for 循环，FBM 可改为通用八度计数
- **碰撞/路径数据**：从 bitmask 中提取可行走区域信息
- **运行时编辑**：通过 PML 的 MCP 接口实现实时地图编辑
