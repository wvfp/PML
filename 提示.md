# PML 纹理映射 Bug 排查提示

## 问题现象

运行以下 4 个示例，渲染出的 PNG 图像中纹理映射效果不正确：

```
examples/13-texture/01_texture_basic.pml
examples/13-texture/02_uv_modes.pml
examples/13-texture/03_wrap_modes.pml
examples/13-texture/04_path_texture.pml
```

具体现象：
- **01_texture_basic**: 星空纹理的星星散布在整个画布，而不是限制在圆角矩形（rx=16）内
- **02_uv_modes**: Planar/Harmonic/Explicit 三种 UV 模式都没有纹理填充，只显示透明轮廓
- **03_wrap_modes**: 三种 wrap 模式（Clamp/Repeat/Mirror）都没有纹理填充
- **04_path_texture**: 彩虹纹理未正确映射到矩形上

## 预期行为

`(texture-map (rect 40 40 320 320 :rx 16) :source starry-sky :mode :harmonic)` 应该在画布 (40,40)-(360,360) 位置渲染一个带星空纹理的圆角矩形。

## 纹理映射 API 流程

```
PML 源 → (define-texture name (w h) go-expr)  → TextureBox
         → (texture-map shape :source tex :mode m ...) → GraphicObject with ParamKey::uv
         → render → draw_object() → draw_textured_object()
```

### 关键文件

| 文件 | 作用 |
|------|------|
| `src/pml/evaluator/texture_builtins.cpp` | `define-texture` 特殊形式，创建 TextureBox |
| `src/pml/evaluator/texture_map_builtins.cpp` | `texture-map` builtin，设置 UV 参数到 GraphicObject |
| `src/pml/graphics/triangulation.cpp` | 调用 poly2tri 进行形状三角化 |
| `src/pml/graphics/uv_solver.cpp` | Planar/Harmonic/Explicit UV 求解 |
| `src/pml/backend/skia/textured_draw.cpp` | **核心：** 烘焙纹理 + 构建 SkVertices + drawVertices |
| `src/pml/backend/skia/skia_backend_draw.cpp` | `draw_object()` 分派，检测 ParamKey::uv 后调用 draw_textured_object |

### draw_textured_object 流程

1. 从 `obj.params.find(ParamKey::uv)` 获取 TextureBox
2. 确定 UV 模式（uv_mode: 0=planar, 1=harmonic, 2=explicit）
3. `triangulate_shape(obj)` → 返回 TriangulatedMesh（vertices + indices + contour_map）
4. 根据模式求解 UV：`solve_planar_uv` / `solve_harmonic_uv` / `apply_explicit_uv`
5. `bake_texture(tex)` → 将 TextureBox 中的 GraphicObject 栅格化为 SkImage
6. 构建 `SkVertices::MakeCopy(kTriangles, positions, texCoords, colors, indexCount, indices)`
7. `canvas->drawVertices(vertices.get(), SkBlendMode::kModulate, paint)`
   - paint 设置 shader = img->makeShader(tile_x, tile_y, sampling)
   - paint 设置 blend mode = kModulate

## 可能的问题方向

### 1. drawVertices 在 Skia Raster 后端的兼容性

Skia 的 `SkSurfaces::Raster`（CPU 软件渲染）对 `drawVertices` 的支持可能有限，特别是：
- `SkBlendMode::kModulate` 在 raster 后端的行为
- `drawVertices(const SkVertices*, SkBlendMode, const SkPaint&)` 的重载选择

可尝试：改用 `drawImageRect` 或 `drawImage` + 矩阵作为替代验证。

### 2. SkVertices 构建方式

当前代码在 `textured_draw.cpp` 中：
```cpp
const size_t n_tri = mesh.indices.size() / 3;
std::vector<SkPoint> tri_pos; tri_pos.reserve(n_tri * 3);
std::vector<SkPoint> tri_uv;  tri_uv.reserve(n_tri * 3);
for (size_t t = 0; t < n_tri; ++t) {
    for (int k = 0; k < 3; ++k) {
        uint32_t idx = mesh.indices[t * 3 + k];
        tri_pos.push_back({mesh.vertices[idx].x, mesh.vertices[idx].y});
        tri_uv.push_back({uv_result.uvs[idx].x, uv_result.uvs[idx].y});
    }
}
auto vertices = SkVertices::MakeCopy(SkVertices::kTriangles_VertexMode,
    tri_pos.size(), tri_pos.data(), tri_uv.data(), nullptr);
```
注意：没有传递 indexCount 和 indices（用了默认 0/nullptr），因为 tri_pos 已按三角形展开。
但也可以考虑改为传递索引而不展开。

### 3. 坐标系/变换

`draw_textured_object` 在 `draw_object` 的 `SkAutoCanvasRestore` 作用域内被调用，此时画布已应用了对象的变换矩阵。但 rect 的位置(40,40)来自参数（ParamKey::x/y），而非变换矩阵，所以变换是单位矩阵，坐标应该是正确的。

### 4. 三角化对带圆角(rx)的 rect 的处理

检查 `triangulation.cpp` 中 `collect_contour("rect")` 是否处理了 rx/ry 参数。当前只返回 4 个角点，没有细分圆角边，可能导致 UV 映射失真。
